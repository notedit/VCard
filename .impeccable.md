## Design Context

### Users
vCard serves creators, operators, educators, and knowledge workers who need to turn news, insights, or explainers into publishable social media card decks for 小红书 / 微信小绿书-style platforms. They work in a desktop web studio and need a fast path from topic to editable outline to export-ready cards.

### Brand Personality
Minimal, professional, editorial. The interface should feel calm, precise, and information-forward rather than playful, decorative, or generic SaaS.

### Aesthetic Direction
Black, white, and grayscale with one sharp accent color. The product should feel like an editorial production desk: structured controls, dense but readable information, serif display type for card content, restrained UI typography for the workspace, and no old hand-drawn wireframe language.

### Design Principles
- Keep the first screen as the actual creation workflow, not a marketing page.
- Prefer a five-step workspace: input, outline, style, generate, studio.
- Make every generated card editable, with chat editing locked to the selected card by default.
- Keep API boundaries clear: the web app works locally now, while generation and image APIs can be swapped in later.
- Use emphasis sparingly: one accent color per screen, no decorative gradients or oversized UI flourishes.
