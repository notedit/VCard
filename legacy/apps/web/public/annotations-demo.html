Annotation Skill matrix

视觉注意力控制系统

静态卡片是信息，带标注的卡片是「内容 + 节奏 + 引导」。下面是 12 种把卡片转成可传播资产的标注效果，每个都在自动循环演示。

用户看哪里什么时候看为什么停留
A

教育类

把静态知识卡 → 讲解视频
01

知识拆解卡片

"3 步理解贝叶斯定理"

1定义先验
2更新观察
3得到后验
config
{
  "type": "step_annotation",
  "mode": "sequential",
  "steps": [
    {"target": "Step1", "effect": "box_highlight"},
    {"target": "Step2", "effect": "arrow_flow"},
    {"target": "Step3", "effect": "zoom_focus"}
  ]
}

逐步讲解 = AI 自动生成课程内容的核心动作

02

错题讲解 / 对比解析

"这个写法为什么错？"

✕== '0'
VS
✓=== '0'
config
{
  "type": "compare_annotation",
  "left": "wrong_example",
  "right": "correct_example",
  "effects": ["shake", "cross_mark", "highlight"]
}

冲突感 → 教育内容点击率最高的母模板

03

公式 / 重点记忆强化

荧光笔 + 结论 glow

P(A|B) = P(B|A)·P(A) / P(B)
→ 后验 ∝ 似然 × 先验
config
{
  "type": "memory_anchor",
  "highlight": "formula",
  "conclusion_effect": "glow"
}

把"看过"变成"记住"，短视频刷屏的隐形机制

B

资讯 / 内容营销

把信息变成观点
04

热点新闻解读

"OpenAI 又出新模型，这次变了什么？"

这次更新的核心不是能力变强，而是交付方式。从"给建议"进入"委托执行"阶段。

config
{
  "type": "opinion_annotation",
  "targets": ["核心观点", "结论句"],
  "style": "editorial"
}

观点比信息更容易被转发

05

config

数据解读

C

产品 / 商业转化

直接影响转化率
07

产品卖点讲解

"这个 AI 产品 3 个核心能力"

语音克隆低成本
低延迟响应<200ms
多语言切换12 种
config
{
  "type": "feature_annotation",
  "features": [
    {"target": "语音克隆", "label": "低成本"},
    {"target": "低延迟", "label": "<200ms"}
  ]
}

逐点高亮 + 价值标签 = 转化路径

10

局部放大（视觉引导）

2x lens + blur background

在一段密集的资讯里，真正决定阅读节奏的从来不是字号，而是「视觉重心」。让读者在该停的地方停下来，是编辑部的技术活。

config
{
  "type": "magnifier",
  "zoom": 1.8
}

非常适合视频里"局部细节"的强引导

11

多卡片转视频

202 / 03能力变化
303 / 03对内容人意味着什么
config
{
  "type": "multi_card_video",

  "annotation": true

12

讲解式视频

类似老师讲课，配合 TTS

先给一个判断：这次更新是范式转折。 原来你给建议，现在你委托执行。 读 → 改 → 跑测试 → 出 PR。 所以"提示词"不再是输入的全部。
config
{
  "type": "lecture_mode",
  "sync_with_voice": true
}

同步逐句高亮 + TTS = 杀手级讲解视频
