01 背景与机会

市场观察

  • 小红书：图文笔记是社区主战场。爆款依赖"封面 + 多图叙事 + 情绪 Payoff"的固定结构，但创作者每次都从零搭。
  • 小绿书（公众号封面卡片）：核心痛点是"图里要有正经汉字"——传统模型出中文经常糊、错字、不工整；用户被迫先生成图再用 Photoshop / Canva 后期合成。
  • Gamma 社媒已经验证"AI 一键生成多卡"的市场，但其设计是 PPT 思维的延伸：通用大纲 → 模板 → 出图，没有针对中文社媒做特化。

机会

把"社媒发什么、怎么发"显式建模为可组合的 AI 能力（Skills），用 gpt-image-2 解决中文文字 + 多图一致性这两个工程难题，做出一个 不是 Gamma 翻译版 的产品。

02 产品定位

一句话

一个 AI native 的社媒卡片工作台 —— 你给主题，它出爆款。

三个原则

PRINCIPLE 01

平台原生

一个平台一套规则。视觉、字数、爆款结构由"目标平台"驱动，不是先选模板再套主题。

PRINCIPLE 02

Agent 优先

大事 Plan，小事 ⌘K。Agent 是动词不是图标，主动评审、可逆建议。

PRINCIPLE 03

Skills 可叠

能力像乐高。"标题手 + 摄影师 + 数据控"自由组合，按优先级生效。

非目标

  • 不做通用图文编辑器（让位给 Figma / Photoshop）。
  • 不做素材市场或图库（gpt-image-2 直出，不依赖现成素材）。
  • 不做 PPT / 长文档（让位给 Gamma / Notion AI）。

05 核心用户旅程

  1. 选平台 · 小红书 / 小绿书
  2. 输主题 + 挂载 Skills（可选）
  3. Agent 生成 Plan — 大纲 + 每张卡的叙事角色
  4. 对话改 Plan — 用户和 Agent 讨论，可拖拽重排、合并、拆分
  5. 选模板 / 风格 — 视觉系统从挂载的 Skill 推断默认值
  6. 视觉生成 — gpt-image-2 出图，主体一致，文字直出
  7. 编辑器精修 — ⌘K 自然语言改写 + Agent 评审
  8. 导出 / 直发 — 分平台尺寸打包，或一键发到对应平台

关键：Skills 在 2-7 全程参与——不是只在第 2 步选一次。

06 功能 1：入口（Home）

需求

  • 顶部 Hero：「今天想发到哪里？」+ 平台卡片 5 个（一期：小红书 + 小绿书可点；其余打灰加"即将推出"）。
  • 主输入区：主题输入框 + Skills 挂载条 + 参数下拉（卡片数 / 尺寸 / 设计模式 / 语言 / 语气）。
  • 下方 4-8 个"今日灵感"卡片，由后端按用户历史 + 平台热门组合推荐，可"换一组"。

交互细节

  • 选平台后，参数下拉会自动调整可选项（小红书默认 4:5 / 9 张；小绿书默认 1:1 / 1 张）。
  • 挂载 Skill 会改变下方"今日灵感"的内容（实时刷新）。
  • 主题输入支持粘贴 URL → 自动抓取并提炼主题；支持参考图 → 进入主体一致性。
  • 「灵感骰子」按钮：从挂载的 Skill 上下文随机生成主题。

验收

  • P0：选平台 → 参数自动调整正确（覆盖 12 种参数组合）
  • P0：URL 粘贴抓取成功率 ≥ 85%（小红书 / 知乎 / 公众号 链接）
  • P1：今日灵感按 Skill 切换平均刷新延迟 < 800ms

07 功能 2：Plan（AI 大纲）

需求

Plan 是这个产品最重要的差异化页面。大纲不是终点，是一个对话现场。

叙事角色 (Role)

每张卡自动打上一个叙事角色： 封面 钩子 立论 清单 Payoff CTA

  • 角色由挂载的 Skill 决定（如「小红书种草体」会强制有 Hook 和 CTA）。
  • 角色显式化的好处：新人理解爆款结构，老手能用 ⌘K 直接说"把第 5 张换成 Payoff"。

左栏：可拖大纲

  • 拖拽节点改顺序（自动通知 Agent 重新衔接上下文）。
  • 点击节点 → 进入卡片局部编辑（弹出文本 + 参考图）。
  • 右键 / ⋯：拆分 / 合并 / 删除 / 让 AI 改写 / 锁定（不允许 Agent 再动）。
  • 追加卡片支持 NLU："让 AI 在 #5 后插入「番外：胡同夜景」"。

右栏：Agent 对话

  • 每次 Plan 生成会附上"策略说明"（为什么这么排）。
  • 用户改一处，Agent 主动提"我连带改了 #2，要保留吗" + 保留 / 撤回 双向操作。
  • 建议气泡有 3 类色：黄=结构 / 蓝=平台 SOP / 紫=技术。

验收

  • P0：拖拽重排后，下游卡片自动连贯衔接（Agent 在 3s 内回写）
  • P0：撤回操作 100% 可逆
  • P1：Agent 主动建议平均"采纳率" ≥ 35%

08 功能 3：Skills（能力包）

定义

Skill = Prompt + 参考图 + 输出约束 + 工具权限。一个 Skill 让 AI 模拟一个特定的"创作者人格"。

Skill Schema（v1）

{
  "id": "skill_redbook_grass",
  "name": "小红书种草体",
  "author": "@小红书官方",
  "category": ["文案", "语气"],
  "system_prompt": "你是小红书顶流种草博主...",
  "few_shot_examples": [...],
  "image_refs": ["ref_url_1", "ref_url_2"],
  "output_schema": {
    "must_have": ["hook_card", "cta_card"],
    "max_words_per_card": 80,
    "title_emoji_prob": 0.7
  },
  "applies_to": {
    "platforms": ["redbook"],
    "stages": ["plan", "edit", "image_prompt"]
  },
  "stack_priority": "high"
}

挂载与叠加

  • 每个项目最多挂 5 个 Skill，按优先级排序（数字 1=最强）。
  • 叠加规则：高优先级 Skill 的 system_prompt 在前；冲突字段以高优先级为准；图像 refs 取并集。
  • "预览叠加效果"按钮：让 Agent 用当前 Skill 组合在样本主题上跑一次小 Plan，秒看风格。

市场（Skills Marketplace）

  • 分类：文案/语气 · 视觉/风格 · 图像生成 · 数据/图表 · 叙事结构 · 平台原生。
  • 排序：本周热门 / 我关注的人 / 与当前项目最匹配。
  • 每个 Skill 卡片含 4 张样图条带（即时预览）+ 下载量 + 标签。
  • 自建 Skill：上传几张参考图 + 写一段说明 + 给几个范例文案，AI 自动生成 schema。

验收

  • P0：5 个 Skill 叠加生效正确（A/B 测试体感差异显著）
  • P1：自建 Skill 完成时间 ≤ 5 分钟
  • P2：Skill 分享链接 + 二维码

09 功能 4：视觉（gpt-image-2 集成）

三个独有能力

CAPABILITY 01

主体一致性锁

左栏的 1-3 张参考图作为锚点。AI 自动提取「光线 / 相机 / 人物 / 道具」，9 张图共享 prompt 前缀，可逐项锁定。

CAPABILITY 02

中文文字烧入

由 gpt-image-2 直接出"图里有中文字"，无需后期。提供 4 种位置模板：顶部居中 / 书法竖排 / 满屏大字 / 底栏字幕。

CAPABILITY 03

局部重生 + 蒙版

悬停单卡 → 局部重生 / 蒙版重绘 / 替换主体；其余 8 张不动，保持视觉一致性。

布局

  • 左栏 (260px)：主体锚点 / 艺术风格 / 文字渲染 设置。
  • 中栏 (3×3 网格)：9 张图实时预览，悬停弹出局部操作。
  • 右栏 (240px)：当前选中卡的 prompt（含全局继承）+ 历史版本（最多 5 版）。

关键约束

  • 整组生成失败时回退：保留已成功的，只重生失败的。
  • 每张图 prompt = 全局 prefix + Skill 风格 token + 本卡特定描述，三段拼接，用户可逐段查看。
  • 文字烧入失败检测：用 OCR 校验生成图里的文字 ≥ 95% 准确率，失败则自动重试 1 次。

验收

  • P0：9 张图同主体一致性主观评分 ≥ 4/5（盲评）
  • P0：中文文字烧入 OCR 准确率 ≥ 92%
  • P0：单图重生不影响其余 8 张（缓存命中）

10 功能 5：编辑器

布局

  • 左栏：卡片层（缩略图 + 标题 + 角色色块）。
  • 中栏：手机内框等比预览，1:1 编辑。点击元素弹出选择框（Title / Body / Image）。
  • 右栏：属性面板（上）+ Agent 建议（下）。

⌘K Command Bar

  • 悬浮在画布底部，永远可达。打开后可：自然语言（"把这张更短"）/ 一键候选（"加 emoji"）/ 模板指令（"按 Skill 重写"）。
  • ⌘ + ↵ 执行；执行结果带差异 diff，确认后才提交。

Agent 建议（右下）

  • 每次编辑后 Agent 异步扫描，最多挂 3 条建议。
  • 建议 = 问题 + 一键修复方案。例：「第 3 / 5 张标题结构相似 → 让 #5 用问句开头？」
  • 建议有"忽略 N 次后此项不再提示"的记忆。

属性面板

  • 文本：字体（受 Skill 约束）/ 字号 / 粗细 / 行高 / 间距。
  • 图像：当前底图缩略 + 历史版本 + 重生 / 蒙版按钮。
  • 布局：仅暴露 3-5 种"位置档"（左上/居中/底栏…），不开放完全自由摆放（避免乱排）。

验收

  • P0：⌘K 平均响应 < 1.2s（首字符）
  • P0：Agent 建议采纳率 ≥ 30%
  • P1：操作历史 Undo/Redo 50 步

11 功能 6：导出 / 直发

  • 导出：ZIP 包，按平台尺寸打包（小红书 4:5 PNG / 小绿书 1:1 PNG / 高清原图 + 压缩版）。
  • 直发（一期 P1）：通过 OAuth 接入小红书 / 公众号开放接口，一键发草稿。
  • 分平台改尺寸：同一组卡片可"导出到另一个平台"——AI 自动改版（4:5 → 1:1 时重新构图）。

12 Agent 行为规范

Voice & Tone

做不做
角色像同事像客服 / 拟人萌宠
开头直接说事"好的，我来帮您…"
意图显式说明"为什么这么改"静默改
结尾给可逆按钮"已完成"硬通知

护栏

  • 从不静默修改用户内容。所有 Agent 改动必须有 diff + 撤回。
  • 同一会话内最多主动提 5 条建议；忽略后降权。
  • 检测平台违规词（如医疗 / 金融极限词）→ 弹平台规则提示，不强删。

14 数据与隐私

  • 用户上传的参考图：仅用于本次生成，不入训练；7 天后清理。
  • Skills 默认私有，"分享"按钮显式同意后才进入市场。
  • 已发布到平台的内容回流：仅记录"我们生成的"内容指标（点赞 / 评论 / 收藏），需用户在平台 OAuth 时同意。

17 风险与开放问题

已知风险

  • gpt-image-2 中文准确率：当前公开数据约 90%。需要内测 OCR 自动重试机制。
  • 主体一致性：人物面部一致仍是难题。M1 默认关闭"锁定人物外观"，避免 demo 翻车。
  • 平台合规：小红书 / 公众号 OAuth 接口可能限频；M3 需提前申请白名单。
  • Skill 滥用：用户上传的参考图可能侵权。需要预审 + 举报机制。

开放问题

  • Q: Skills 是否收费？建议官方 Skill 免费，第三方 Skill 走分成。待法务确认。
  • Q: 导出水印？建议默认无水印（付费版），免费版加角标。
  • Q: Agent 主动评审的"忽略 N 次"阈值是多少？建议 N=3，可调。
  • Q: ⌘K 在移动端如何替代？建议长按卡片 → 弹出底部输入条。
